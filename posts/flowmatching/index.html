Flow models III: Flow Matching

March 2025

In the preceding notes, we've seen how diffusion models are trained and sampled from and we've seen how the score function ∇ln⁡pt\nabla \ln p_t∇lnpt​ is efficiently learnt using score matching. However, since their inception, diffusion models felt a little bit weird for various reasons.

  • First, they did not « really » bridge p0p_0p0​ with N(0,Id)N(0,I_d)N(0,Id​). They bridge p0p_0p0​ with pTp_TpT​ which is only approximately N(0,Id)N(0,I_d)N(0,Id​). This is absolutely not important practically, but from a theoretical point of view, it is a bit unsatisfactory. There should be a way to bridge p0p_0p0​ with N(0,Id)N(0,I_d)N(0,Id​) exactly in finite time.

  • Second, the design of a diffusion feels a little bit clunky. How do we choose the drift and diffusion coefficients μt,wt\mu_t, w_tμt​,wt​? In the end, it looks that the coefficients αt,σt\alpha_t, \sigma_tαt​,σt​ such that xtx_txt​ has the same law as αtx0+σtε\alpha_t x_0 + \sigma_t \varepsilonαt​x0​+σt​ε are the ones who matter, so why note directly choose them?

  • Finally, this ODE/SDE duality is a bit confusing. In the end, the SDE formulation is not really useful since we are only interested in the marginals ptp_tpt​, and the ODE sampling feels really simpler. There was a time when it was not clear why SDEs seemed to work better (while there was no theoretical reason for that).

For these reasons, the community has been looking for a new model that would be more intuitive, more flexible, and more powerful, able to bridge any two distributions in finite time, with deterministic (ODE) sampling. In the end, it turns out that Flow-Matching is almost entirely equivalent to diffusion score matching, except that the presentation and the way we're doing things is slightly different – but way more flexible.

Flow matching

Let (X0,X1)(X_0, X_1)(X0​,X1​) be a couple of random variables sampled from p0p_0p0​ and p1p_1p1​. Actually, they can even be dependent: we only impose that their marginal distributions are p0p_0p0​ and p1p_1p1​, so we note π\piπ their joint distribution and we suppose that ∫π(x,y)dx=p1(y)\int \pi(x,y)dx = p_1(y)∫π(x,y)dx=p1​(y) and ∫π(x,y)dy=p0(x)\int \pi(x,y)dy = p_0(x)∫π(x,y)dy=p0​(x). From a probabilistic perspective, (X0,X1)(X_0, X_1)(X0​,X1​) can be any coupling between p0,p1p_0, p_1p0​,p1​.

Sampling the probability path.

Sampling XtX_tXt​ is easy when we have at our disposal samples X0,X1X_0, X_1X0​,X1​ from p0,p1p_0, p_1p0​,p1​. But when we have only one of them, say X0X_0X0​, we cannot use this formula, so we have to sample the ODE x˙t=vt(xt)\dot{x}_t = v_t(x_t)x˙t​=vt​(xt​) started at X0X_0X0​: but this would need knowledge of vtv_tvt​. That is not directly doable, since its expression needs knowledge of ptp_tpt​. However, the L2L^2L2 loss E[∣s(Xt)−vt(Xt)∣2]\mathbb{E}[|s(X_t) - v_t(X_t)|^2]E[∣s(Xt​)−vt​(Xt​)∣2] can efficiently be minimized without knowing vtv_tvt​.

Learning the velocity

Flow Matching Loss.

Let sss be any function. Then,

E[∣s(Xt)−vt(Xt)∣2]=E[∣s(Xt)−X˙t∣2]+c,\mathbb{E}[|s(X_t) - v_t(X_t)|^2] = \mathbb{E}[|s(X_t) - \dot{X}_t|^2] + c,E[∣s(Xt​)−vt​(Xt​)∣2]=E[∣s(Xt​)−X˙t​∣2]+c,

where c=E[∣X˙t∣2]−E[∣vt(Xt)∣2]c = \mathbb{E}[|\dot{X}_t|^2] - \mathbb{E}[|v_t(X_t)|^2]c=E[∣X˙t​∣2]−E[∣vt​(Xt​)∣2] is a constant with respect to sss.

The practical consequence is that if sθs^\thetasθ is smoothly parametrized by θ\thetaθ, then the L2L^2L2-loss

L⋆(θ)=E[∣sθ(Xt)−vt(Xt)∣2]L_\star(\theta) = \mathbb{E}[|s^\theta(X_t) - v_t(X_t)|^2]L⋆​(θ)=E[∣sθ(Xt​)−vt​(Xt​)∣2]

and the Flow-Matching loss

L(θ)=E[∣sθ(Xt)−X˙t∣2]L(\theta) = \mathbb{E}[|s^\theta(X_t) - \dot{X}_t|^2]L(θ)=E[∣sθ(Xt​)−X˙t​∣2]

have the same gradients and the same minimizers.

Proof. Develop the square: E[∣s(Xt)−vt(Xt)∣2]=E[∣s(Xt)∣2]+E[∣vt(Xt)∣2]−2E[⟨s(Xt),vt(Xt)⟩].\begin{aligned}\mathbb{E}[|s(X_t) - v_t(X_t)|^2] &= \mathbb{E}[|s(X_t)|^2] + \mathbb{E}[|v_t(X_t)|^2] - 2\mathbb{E}[\langle s(X_t), v_t(X_t)\rangle]. \\ \end{aligned}E[∣s(Xt​)−vt​(Xt​)∣2]​=E[∣s(Xt​)∣2]+E[∣vt​(Xt​)∣2]−2E[⟨s(Xt​),vt​(Xt​)⟩].​ The last term is equal to

Design of conditional flows

Now that everything is set, we have to design an efficient conditional flow φt\varphi_tφt​.

Linear flows

The simplest (and, indeed, very powerful) flow is the linear one, φt(x,y)=αtx+σty\varphi_t(x,y) = \alpha_t x + \sigma_t yφt​(x,y)=αt​x+σt​y, giving

Xt=αtX0+σtX1.X_t = \alpha_t X_0 + \sigma_t X_1.Xt​=αt​X0​+σt​X1​.

Here, α,σ\alpha, \sigmaα,σ are differentiable functions and must satisfy the constraint α0=σ1=1\alpha_0 = \sigma_1 = 1α0​=σ1​=1 and α1=σ0=0\alpha_1 = \sigma_0 = 0α1​=σ0​=0. The trajectories are straight lines going from X0X_0X0​ to X1X_1X1​ at a velocity given by

X˙t=α˙tX0+σ˙tX1.\dot{X}_t = \dot{\alpha}_t X_0 + \dot{\sigma}_t X_1.X˙t​=α˙t​X0​+σ˙t​X1​.

In the simplest setting, we can simply choose αt=1−t\alpha_t =1-tαt​=1−t and σt=t\sigma_t = tσt​=t. This is called rectified flows. The velocity is constant, X˙t=X1−X0\dot{X}_t = X_1 - X_0X˙t​=X1​−X0​, so that the flow-matching loss minimizes E[∣s(Xt)−(X1−X0)∣2]\mathbb{E}[|s(X_t) - (X_1 - X_0)|^2]E[∣s(Xt​)−(X1​−X0​)∣2].

"Gaussian" flows

Formula (16) tells us (once again!) that the only thing that matters given the choice of αt,σt\alpha_t, \sigma_tαt​,σt​ is the knowledge of the score ∇ln⁡pt\nabla \ln p_t∇lnpt​. How this score was learnt is actually an independent problem: we don't care how the learning was done. When we have the score, we can plug it in (16) and sample.

By carefully looking at the derivations, we can see that there are one-to-one linear connections between

  • ∇ln⁡pt\nabla \ln p_t∇lnpt​ (the score)

  • E[ε∣Xt]\mathbb{E}[\varepsilon \mid X_t]E[ε∣Xt​] (the denoising model)

  • E[X0∣Xt]\mathbb{E}[X_0 \mid X_t]E[X0​∣Xt​] (the data prediction model)

  • vtv_tvt​ (the velocity model).

Only one of them needs to be learned, and we can convert it into the three others. If one wants to use a pretrained model for sampling, one thus needs to keep track of how the model was learned: is it a score, a denoiser, a data predictor or a velocity?

We close this part with an important note: given a score model for ∇ln⁡pt\nabla \ln p_t∇lnpt​, regardless of how it is formulated and trained, using it to sample from the flow x˙t=vt(xt)\dot{x}_t = v_t(x_t)x˙t​=vt​(xt​) or to sample from the DDIM ODE in diffusion models is exactly the same.

Brownian bridges

Optimal transport flows

There is room for the choice of the connection φt\varphi_tφt​; among these choices, some of them should be better than the others. The choice of one of these connections provides a velocity field vtv_tvt​ transporting samples from p0p_0p0​ to samples from p1p_1p1​. Consider all the possibles fields utu_tut​ such that the solutions of the ODE x˙t=−u(xt)\dot{x}_t = -u_(x_t)x˙t​=−u(​xt​) started at x0∼p0x_0 \sim p_0x0​∼p0​ and at x1∼p1x_1 \sim p_1x1​∼p1​. The best possible one should minimize the total kinetic energy,

E∫01∣ut(xt)∣2dt\mathbb{E}\int_0^1 |u_t(x_t)|^2dtE∫01​∣ut​(xt​)∣2dt

where the expection is over p0p_0p0​. This problem is widely studied and can be solved: it is quite intuitive that the optimal flows are straight lines, so the optimal velocity fields are constants, and indeed the flows are given by ut⋆(x)=x+t(ϕ(x)−x)u^\star_t(x) = x+t(\phi(x)-x)ut⋆​(x)=x+t(ϕ(x)−x) where φ\varphiφ is Brenier's potentiel, the unique (under some conditions) map ϕ:Rd→Rd\phi : \mathbb{R}^d \to \mathbb{R}^dϕ:Rd→Rd such that ϕ(X0)\phi(X_0)ϕ(X0​) has distribution p1p_1p1​ and which minimizes the square transport distance E[∣x−φ(x)∣2]\mathbb{E}[|x - \varphi(x)|^2]E[∣x−φ(x)∣2]. Of course, computing ϕ\phiϕ is in general intractable.

This solves the unconditional OT problem, and we need conditional flows. One nice trick goes as follows: using Jensen's inequality and de-conditioning, we can write

The conclusion of these considerations is as follows.

The rectified flow Xt=(1−t)X0+tX1X_t = (1-t)X_0 + tX_1Xt​=(1−t)X0​+tX1​ is a minimizer of a bound on the Kinetic Energy among all the flows transporting p0p_0p0​ to p1p_1p1​.

It might not be a minimizer of the Kinetic Energy itself, but it is a good starting point. In any ways, it is important to keep in mind that flowing straight between two points is absolutely not the same as flowing straight between two distributions.

Conclusion

Up to now, we've seen how flow matching reformulates and simplifies diffusion sampling. There remains a problem: in practive, to sample from these models, we need to solve an ODE or a SDE, which in practive is done by discretizing time and using a scheme like Euler or Runge-Kutta. But for every time step, this needs a feedforward evaluation of a neural network. Most schemes need 50 time steps, which is why sampling from diffusions can be long.

Consistency models try to directly learn the flow φt\varphi_tφt​ and not the velocity vtv_tvt​ - and yes, the naming should have been velocity matching from the beginning.
